# Audio UI / UX

Use this skill when an audio tool works technically but feels awkward, confusing, or slow.

## Core rule

Audio UI should reveal sound, not hide it.

## Good defaults

- most important controls first
- obvious gain staging
- readable units
- fast A/B testing
- clear bypass
- meaningful presets/categories

## Common mistakes

- too many knobs with vague names
- no visual clue what changed
- meters that look cool but say nothing
- tiny hit targets
- modulation hidden in weird submenus
- “character” controls with no mental model

## Naming

Prefer names musicians understand:
- tone
- drive
- mix
- width
- attack
- decay
- air
- body

Avoid names that sound clever but explain nothing.

## Visual hierarchy

Show:
- primary sound-shaping controls
- output safety controls
- state/level feedback
- secondary settings later

## Output style

When helping, prefer:
- what should be visible first
- which controls should move or group together
- what a musician expects
- one simplification move
